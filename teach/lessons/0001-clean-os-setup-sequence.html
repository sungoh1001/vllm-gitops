Lesson 0001

Clean OS 이후 작업 순서

2026-06-17 · 소요시간 약 15분 · 연관: GLOSSARY, MISSION, RESOURCES

Ubuntu Server 24.04를 Dell R740xd에 새로 설치한 직후의 상태에서, Docker로 GPU 컨테이너를 실행할 수 있기까지의 작업 순서와 각 단계에서의 판단 기준을 다룹니다.

이 수업이 끝나면 여러분은:

  • 각 설치 단계가 왜 그 순서여야 하는지 설명할 수 있습니다
  • 에어갭 환경에서도 적용 가능한 설치 방법을 압니다
  • 설치가 잘 되었는지 검증하는 테스트를 실행할 수 있습니다
---

1. 현재 상태 확인

알고 있는 상태 OS 설치 완료 · NVIDIA 드라이버 설치 완료 (nvidia-smi 동작) · git 설치됨 · docker 미설치

nvidia-smi로 확인할 항목들: 

$ nvidia-smi
$ nvidia-smi topo -m    # GPU 토폴로지: PCIe 경로 확인
$ git --version           # git 존재 확인
$ docker --version        # "command not found" 예상됨
GPU 토폴로지 확인 이유: PCIe 브릿지, GPU-RAID 컨트롤러 간 거리, NUMA 노드 바인딩을 미리 파악해야 Docker로 GPU를 할당할 때 최적 경로를 선택할 수 있습니다. 특히 A100 2장이 서로 다른 NUMA 노드에 있는지 확인하세요.
---

2. 전체 작업 파이프라인

Clean OS에서 GPU 컨테이너 실행까지의 흐름입니다:

① APT 미러 설정 → ② Docker Engine 설치 → ③ NVIDIA Container Toolkit 설치 → ④ Docker GPU 런타임 설정 → ⑤ Private Registry 설정 → ⑥ GPU 컨테이너 테스트
순서가 중요한 이유: ③ NVIDIA Container Toolkit은 ② Docker Engine이 먼저 설치되어 있어야 합니다(NVIDIA Container Toolkit이 Docker의 런타임 경로에 자신을 등록하기 때문). 그리고 ⑥ 테스트는 ④⑤ 설정이 완료된 후에야 통과합니다. 순서를 바꾸면 재설치를 해야 할 수 있습니다.
---

3. 각 단계 상세

  1. APT 미러 설정

    Ubuntu 24.04는 APT 소스 파일이 /etc/apt/sources.list.d/ubuntu.sources (DEB822 형식)로 관리됩니다. 내부망 APT 미러 주소로 변경합니다.

    $ sudo vim /etc/apt/sources.list.d/ubuntu.sources
# 내부 미러 URL로 URIs 변경
$ sudo apt update

    Ubuntu 공식 미러 가이드 (참고용)

  2. Docker Engine 설치

    에어갭 환경에서는 Docker 공식 apt repo 대신, 패키지를 내부 미러에서 받아 설치합니다.

    $ sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin
$ sudo usermod -aG docker $USER          # sudo 없이 docker 명령 사용
$ docker --version

    🔗 주 출처: Docker 공식 — Ubuntu 설치 가이드

    sudo 없이 docker 실행하기: usermod -aG docker 후에 로그아웃 후 재로그인해야 그룹 변경이 적용됩니다. 재부팅 없이 적용하려면 newgrp docker 또는 su - $USER.
  3. NVIDIA Container Toolkit 설치

    NVIDIA 드라이버만으로는 Docker가 GPU에 접근할 수 없습니다. NVIDIA Container Toolkit이 드라이버와 Docker 사이의 다리 역할을 합니다.

    에어갭 환경: 내부 APT 미러에 nvidia-container-toolkit 패키지가 미러링되어 있어야 합니다. 없다면 관리자에게 요청하거나, .deb 파일을 수동으로 가져와 dpkg -i로 설치해야 합니다.

    $ sudo apt install nvidia-container-toolkit

    🔗 주 출처: NVIDIA Container Toolkit 공식 설치 가이드

    일반적인 실수: 드라이버(nvidia-driver-550)가 설치되어 있으면 GPU가 보이니까, Toolkit 없이도 Docker에서 GPU가 될 거라고 생각하기 쉽습니다. 안 됩니다. Toolkit 없이 docker run --gpus all을 하면 "could not select device driver" 오류가 납니다.
  4. Docker GPU 런타임 설정

    NVIDIA Container Toolkit을 Docker의 런타임으로 등록합니다. 이 명령 한 줄로 /etc/docker/daemon.json에 NVIDIA 런타임이 자동 등록됩니다.

    $ sudo nvidia-ctk runtime configure --runtime=docker
$ sudo systemctl restart docker

    설정 후 /etc/docker/daemon.json을 확인하면 "runtimes": {"nvidia": {...}} 항목이 추가된 것을 볼 수 있습니다.

  5. Private Registry 설정

    에어갭 환경에서는 모든 Docker 이미지를 내부 Registry에서 pull 받아야 합니다. /etc/docker/daemon.json에 registry mirror를 추가합니다.

    {
  "registry-mirrors": ["https://nexus.internal.company.com"],
  "runtimes": {
    "nvidia": { ... }
  }
}
    $ sudo systemctl restart docker

    Nexus, Harbor 등 회사에서 사용 중인 Registry 주소로 설정하세요.

  6. GPU 컨테이너 실행 테스트

    가장 작은 CUDA 이미지로 GPU 접근을 검증합니다. nvidia-smi가 컨테이너 안에서도 동일하게 보여야 합니다.

    $ docker run --rm --gpus all nvidia/cuda:12.6-base nvidia-smi

    출력에 A100 80GB × 2장이 보이면 성공입니다. 만약 이미지가 내부 Registry에 없다면 미리 docker pull 받아두세요.

    검증 체크리스트:
    ✅ 컨테이너 안에서 nvidia-smi가 GPU 2장을 표시하는가
    ✅ 각 GPU의 메모리가 80GB로 인식되는가
    ✅ 드라이버 버전이 호스트와 동일한가
    ✅ --gpus '"device=0"'로 GPU 1장만 선택 가능한가
